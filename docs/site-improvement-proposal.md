# 開邦高校雄飛会 公式サイト改善素案

> 作成日: 2026-02-22
> 作成者: 神谷乗治（14期）
> ステータス: ドラフト（上間さん・運営メンバーレビュー待ち）

---

## 1. 現状分析と課題

### 1.1 現在のサイト構成

| 項目 | 現状 |
|------|------|
| フレームワーク | Nuxt 3 + Tailwind CSS |
| ホスティング | GitHub Pages（無料） |
| ドメイン | kaiho-yuuhikai.jp |
| サイトタイトル | 「沖縄県立 開邦高校 第三回 大同窓会」 |
| 主要ページ | Home / News / Members / Sponsors / Career / Privacy |
| データ管理 | JSON（cms-data.json）による静的CMS |

### 1.2 課題

1. **サイトが「第3回大同窓会」イベント特化になっている** — イベント終了後、定常的な組織サイトとしての機能を果たしていない
2. **雄飛会の組織情報が不在** — 理念、方針、規約、役員名簿などの常設コンテンツがない
3. **新事業のプラットフォームがない** — メンター制度、基金、議案募集など新たな活動を載せる場所がない
4. **「ワクワク感」の不足** — 現サイトはミニマルで洗練されているが、沖縄セルラー感謝祭のようなエネルギッシュな雰囲気が薄い
5. **外部コンテンツとの連携が未整備** — note.com、Google Forms、Google Docsとの導線がない
6. **支部管理の仕組みがない** — 東京・海外・関西支部のメンターや名簿管理

## 2. コンセプト：「イベントサイト」→「組織プラットフォーム」への転換

### 2.1 基本方針

```
【Before】大同窓会の告知・集客サイト
    ↓
【After】雄飛会の活動拠点 + 大同窓会アーカイブ + 新事業プラットフォーム
```

### 2.2 参考サイト分析（沖縄セルラー感謝祭）

沖縄セルラー感謝祭サイトから取り入れるべき要素：

| 要素 | 沖縄セルラーでの実装 | 雄飛会への応用 |
|------|---------------------|---------------|
| 固定ナビゲーション | スクロール追従型4項目メニュー | セクション別スムーススクロール |
| ビジュアルインパクト | 大型ヒーロー画像 + カテゴリ別アイコン | 開邦の校舎・イベント写真 + 活動カテゴリアイコン |
| カテゴリ分類 | 体験を4種アイコンで視覚分類 | 活動を「基金」「メンター」「イベント」等でアイコン分類 |
| グリッドレイアウト | コンテンツカード2列配置 | 活動・プロジェクトカード表示 |
| CTA設計 | 申込みボタン、特典誘引 | 寄付ボタン、メンター登録、イベント参加 |
| カラースキーム | ビビッドシアンブルー + パープル | 現行のkaiho-green + gold を活かしつつ、アクセントカラー追加 |

### 2.3 デザインの方向性

**現行のミニマルな美しさ** は活かしつつ、以下でワクワク感を追加：

- **ヒーロー領域の動的演出** — パーティクルアニメーション or パララックス効果
- **数字のカウントアップ** — 「卒業生 5,000名以上」「37期の歴史」などの実績をアニメーション表示
- **カード型コンテンツ** — 各活動をカラフルなカードで視覚化
- **人物フォーカス** — メンターや役員の写真を大きく配置し「人が見える」サイトに
- **ステータスバッジ** — 「募集中」「開催予定」「NEW」等のライブ感あるバッジ

#### 決済方式の検討

| 方式 | メリット | デメリット | 推奨度 |
|------|---------|-----------|--------|
| 銀行振込 | 手数料なし、信頼性高 | 手動管理、利便性低 | ◎ 初期は必須 |
| Peatix | 既に大同窓会で実績あり | 手数料あり | ○ イベント紐付き寄付 |
| Syncable | NPO向け寄付プラットフォーム | 団体登録が必要 | △ 中期検討 |
| Stripe埋め込み | カード決済、自動化 | 開発コスト、手数料 | △ 長期検討 |

## 5. 技術実装方針

### 5.1 アーキテクチャ（変更なし・拡張）

```
┌─────────────────────────────────────────────────┐
│                  Nuxt 3 + Tailwind CSS           │
│                  (SSG / GitHub Pages)             │
├─────────────────────────────────────────────────┤
│  composables/                                    │
│    ├── useCmsData.ts      ← 既存拡張             │
│    ├── useMentorData.ts   ← 新規（メンター管理）  │
│    └── useFundData.ts     ← 新規（基金管理）      │
├─────────────────────────────────────────────────┤
│  public/data/                                    │
│    ├── cms-data.json      ← 既存（ニュース等）    │
│    ├── mentors.json       ← 新規（メンター一覧）  │
│    ├── officers.json      ← 新規（役員名簿）      │
│    ├── fund-status.json   ← 新規（基金状況）      │
│    └── branches.json      ← 新規（支部情報）      │
├─────────────────────────────────────────────────┤
│  外部連携                                        │
│    ├── Google Forms       ← iframe 埋め込み       │
│    ├── Google Docs        ← リンク参照            │
│    ├── note.com           ← RSS/oEmbed 埋め込み   │
│    └── Google Sheets      ← データソース（手動変換）│
└─────────────────────────────────────────────────┘
```

### 5.2 データフロー（スプレッドシートCMS方式）

```
Google Sheets（運営メンバーが編集）
    ↓ GitHub Actions or ローカルスクリプト
JSON ファイル（public/data/）
    ↓ useCmsData composable
Vue コンポーネント表示
    ↓ nuxt generate
GitHub Pages（静的HTML配信）
```

- **ランニングコスト: ¥0**（GitHub Pages + Google Workspace 無料枠）
- ドメイン維持費のみ（年間 ¥1,500程度）

### 5.3 Google Sheets → JSON 自動化

```
手順:
1. Google Sheets にメンター情報・役員情報等を管理
2. GitHub Actions の scheduled workflow で定期的に取得
   (google-spreadsheet npm パッケージ or Google Sheets API)
3. JSON に変換して public/data/ に配置
4. nuxt generate で静的サイト再生成
5. GitHub Pages にデプロイ
```

あるいはよりシンプルに:
```
手順（手動フロー — 初期推奨）:
1. Google Sheets を CSV エクスポート
2. scripts/convert_csv_to_json.py で JSON 変換
3. git push → 自動デプロイ
```

## 11. 沖縄セルラー風「ワクワク感」の実装ポイントまとめ

1. **ビジュアルファースト** — テキスト中心 → 写真・イラスト・アイコンを多用

5. **ステータスバッジ** — 「募集中」「NEW」「開催予定」で今動いている感を演出
6. **数字のアニメーション** — 実績を数字で見せ、スクロールで動かす
7. **CTA（行動喚起）の充実** — 各ページに明確なアクションボタン
